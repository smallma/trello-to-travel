Task 1：建立專案骨架與基本 HTML

Files: Create index.html, style.css, app.js

建立 HTML 骨架（Header + Toolbar + Main 區），引入 Tailwind CDN、Noto 字體、marked.js。style.css 定義配色變數、卡片動畫、標籤色票。app.js 暫時只接 file input 事件，console log 確認可運作。

驗收：打開 index.html 看到深綠 Header、可選檔案、console 看到 log。

Commit：feat: scaffold static page with upload UI

Task 2：建立 parser.js（TDD）

Files: Create parser.js, tests.html, tests/parser.test.js

先建立極簡的瀏覽器測試 runner（tests.html 提供 test/assert/assertEq/group 全域函式）。
寫 11 個單元測試覆蓋：

  • 輸入驗證（null、缺欄位 throw）
  • 回傳結構（trip / days / extras / warnings）
  • 日期清單依月日排序
  • 過濾 closed lists / cards
  • 卡片依 pos 升冪
  • 非日期清單分到 extras
  • 欄位映射（name → title, desc, locationName/address → place, labels, attachments）
  • 無日期清單時放 warning

測試先紅 → 寫實作 → 測試全綠。最後用真實 trello.json 在 console 跑一次煙霧測試。

Commit：feat: add parser for Trello board JSON

Task 3：建立 exporter.js（TDD）

Files: Create exporter.js, tests/exporter.test.js

純函式 toCopyJson(data, {includeExtras}) → 回傳格式化 JSON 字串。5 個測試覆蓋：

  • includeExtras=true 時含 extras key
  • includeExtras=false 時不含 extras key
  • 必要欄位：trip, exported_at(ISO 格式), days
  • items 欄位完整保留
  • 輸出格式化（含換行縮排）

Commit：feat: add JSON exporter for LLM consumption

Task 4：建立 renderer.js

Files: Create renderer.js

純 DOM 操作。函式：

  • renderApp(data) — 更新標題 + 渲染每日區塊 + 補充資訊
  • renderDay(day) — 日期區塊頭（可收合）+ 卡片列表
  • renderCard(item) — 卡片標題、第一行預覽、可展開細節區（Markdown 渲染、地點、附件、標籤）
  • renderExtras(extras) — 補充資訊區（預設收起）

所有點擊行為都用 classList.toggle 切換狀態 class，動畫交給 CSS。

Commit：feat: add DOM renderer for parsed data

Task 5：串接 app.js（上傳→渲染→複製）

Files: Modify app.js

把三大模組串起來。事件：

  • 檔案上傳 → 驗副檔名 → 讀取 → parse → render → 顯示 toolbar
  • 全部展開 / 全部收合按鈕
  • 複製按鈕（含 includeExtras 勾選狀態，及 HTTPS / fallback execCommand）
  • 錯誤 banner（紅 / 黃）、底部 toast 提示

手動驗收（清單在 plan）：

  • 真實 trello.json 上傳，所有區塊與卡片正常
  • 展開/收合互動正確
  • 複製出來的 JSON 在編輯器中是合法格式
  • 取消勾選補充資訊後複製，輸出不含 extras
  • 三種錯誤情境（非 JSON / parse 失敗 / 缺欄位）正確顯示 banner

Commit：feat: wire up upload, render, copy flow
